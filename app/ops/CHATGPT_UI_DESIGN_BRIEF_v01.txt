LO QUE FALTA PARA “PRODUCTO COMPLETO” (README / visión)
--------------------------------------------------------------------------------
- Portal paciente, reservas/billing end-to-end, telemedicina, EHR, asistente IA
  multilingüe, facturación/seguros, WCAG 2.1, hardening compliance (Ley 25 Québec,
  PIPEDA, alineación HIPAA). La mayoría no está implementada en UI ni flujos.

FALLOS / FICCIONES REGISTRADAS (sesiones previas; mitigaciones donde aplica)
--------------------------------------------------------------------------------
- Entorno sin Node/npm al inicio → bloqueo de tests hasta nvm/LTS.
- Import supertest CJS → corregido a `import * as request from 'supertest'`.
- Docker socket sin permiso → sudo o grupo docker.
- `rg` ausente en host → scripts usan sed/awk.
- Artefactos de pegado en terminal (^[[200~) mezclando comandos.
- Validación runtime de algunos incrementos: pendiente confirmación local si no se
  pegó salida.

PEDIDO DE DISEÑO (ENTREGABLE QUE QUIERO DE TI, CHATGPT)
--------------------------------------------------------------------------------
1) Propón una arquitectura de información (mapa del sitio / app) que unifique:
   - Superficie pública (marketing mínimo)
   - Acceso paciente / clínico / admin
   - Flujos: registro, login, agenda, sesión (placeholder tele), mensajes, facturación
     (placeholder), registros clínicos (placeholder), ajustes, ayuda/legal
2) Para cada pantalla o vista, indica:

   - Qué debe mostrarse GRIS / disabled / con tooltip “Próximamente” o
     “No conectado aún” sin romper la navegación
3) Reglas de estado visual unificadas (colores, iconos, copy corto EN/FR donde aplique)
   para que un revisor humano entienda qué es real y qué es cascarón.
4) Lista priorizada de implementación en fases (Fase A: shell navegable + gris;
   Fase B: conectar API existente; Fase C: nuevos endpoints).
5) Checklist de accesibilidad mínima (contraste, foco, etiquetas) aunque parte
   esté deshabilitada.
6) Salida en un solo documento continuo (secciones numeradas), listo para que otro
   agente (Cursor) implemente siguiendo el mismo orden.
